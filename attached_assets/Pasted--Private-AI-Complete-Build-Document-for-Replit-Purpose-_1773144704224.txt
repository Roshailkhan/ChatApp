## 1. Product Overview

**Private AI** is a privacy-first AI chat platform. Users get access to multiple LLMs, specialized AI personas (Companions), persistent project workspaces (Spaces), intelligent memory, and powerful research tools — all without any of their data being logged, used for training, or exposed to third parties.

### Core Design Principle
Privacy is not a feature. It is the foundation everything else is built on. Every architectural and product decision must be evaluated through this lens first.

### Product Layers (Bottom to Top)

```
┌─────────────────────────────────────────────────────────────┐
│         COMPANIONS LAYER  (User-Facing Personas)            │
│   Data Analyst | Writer | Researcher | Healthcare |         │
│   Finance | Summarizer | Custom                             │
├─────────────────────────────────────────────────────────────┤
│         TOOLS LAYER  (Capabilities)                         │
│   AI Chat | Code Gen | Deep Research | Web Search           │
├─────────────────────────────────────────────────────────────┤
│         INTELLIGENCE LAYER  (Models)                        │
│   Multi-Model Access: Open Source + Closed APIs             │
├─────────────────────────────────────────────────────────────┤
│         PRIVACY CORE  (Foundation — Never Bypassed)         │
│   E2E Encryption | IP Masking | Zero-Knowledge |            │
│   Local Keys | No Training on User Data                     │
└─────────────────────────────────────────────────────────────┘
```

## 2. Tech Stack

### Frontend
| Layer | Choice | Reason |
|---|---|---|
| Framework | **Next.js 14 (App Router)** | SSR, routing, API routes in one repo |
| Language | **TypeScript** | Type safety across frontend + backend |
| Styling | **Tailwind CSS** | Utility-first, fast to build |
| UI Components | **shadcn/ui** | Accessible, unstyled, composable |
| State Management | **Zustand** | Lightweight global state |
| Data Fetching | **TanStack Query (React Query)** | Server state, caching, background refetch |
| Real-time / Streaming | **Vercel AI SDK (`ai` package)** | Streaming LLM responses, useChat hook |
| Icons | **Lucide React** | Consistent icon set |

### Backend
| Layer | Choice | Reason |
|---|---|---|
| API Runtime | **Next.js API Routes / Route Handlers** | Co-located with frontend |
| Auth | **Clerk** | Managed auth, easy user identity, JWT |
| Encryption | **Node.js `crypto` (AES-256-GCM)** | Encrypt all user data before DB write |
| Key Management | **Per-user encryption keys derived from auth token** | Zero-knowledge: server never holds plaintext |
| Proxy Layer | **Custom fetch wrapper** | Strips IP, headers before external API calls |
| Background Jobs | **Inngest** (or Vercel Cron) | Deep research async jobs |

### Database
| Layer | Choice | Reason |
|---|---|---|
| Primary DB | **PostgreSQL via Neon** | Serverless Postgres, Replit-friendly |
| ORM | **Drizzle ORM** | Type-safe, lightweight, works with Neon |
| Vector Store (optional, P2) | **pgvector** extension on Neon | Semantic memory search |
| Cache | **Upstash Redis** | Session data, rate limiting |

### External AI APIs
| Model | Provider | SDK |
|---|---|---|
| GPT-4o | OpenAI | `openai` npm package |
| Claude 3.5 Sonnet | Anthropic | `@anthropic-ai/sdk` |
| Gemini Pro | Google | `@google/generative-ai` |
| DeepSeek V3 | DeepSeek API | Fetch (OpenAI-compatible) |
| Qwen 2.5 | Together AI or self-hosted | Fetch |
| Mistral 8B / 3B | Mistral API or self-hosted | `@mistralai/mistralai` |

### Web Search
| Tool | Choice |
|---|---|
| Web Search API | **Brave Search API** (privacy-respecting, no tracking) |
| Deep Research | **Tavily API** (research-optimized search) |

## 5. Database Schema

All user-generated content fields (message content, space context, memory facts, companion prompts) are stored **AES-256-GCM encrypted**. The DB schema stores ciphertext, IV, and auth tag, never plaintext.

## 6. Architecture & Request Flow

### Every request goes through this pipeline, no exceptions:

```
User Input
    │
    ▼
┌──────────────────────────────────────────────────┐
│  1. PRIVACY GATE  (lib/privacy/proxy.ts)          │
│     • Strip real IP from request headers          │
│     • Remove device fingerprint headers           │
│     • If PII redaction enabled → run redact.ts    │
│     • Encrypt content with user's derived key     │
└──────────────────────────────────────────────────┘
    │
    ▼
┌──────────────────────────────────────────────────┐
│  2. ROUTING DECISION  (lib/ai/router.ts)          │
│     • Read: user preference, task type,           │
│       cost tolerance, speed requirement           │
│     • Select: open-source path OR closed API path │
│     • Inject: space context + memory + companion  │
│       system prompt into final messages array     │
└──────────────────────────────────────────────────┘
    │                          │
    ▼                          ▼
┌────────────────┐    ┌──────────────────────────┐
│ OPEN-SOURCE    │    │ CLOSED API PATH           │
│ PATH           │    │ (via anonymizing proxy)   │
│ • Mistral 3B   │    │ • GPT-4o → OpenAI API     │
│ • Mistral 8B   │    │ • Claude → Anthropic API  │
│ • Qwen 2.5     │    │ • Gemini → Google API     │
│ • DeepSeek V3  │    │ • DeepSeek → DeepSeek API │
└────────────────┘    └──────────────────────────┘
    │                          │
    └──────────┬───────────────┘
               ▼
┌──────────────────────────────────────────────────┐
│  3. RESPONSE HANDLING                             │
│     • Stream chunks to client via SSE             │
│     • On stream complete:                         │
│       - Encrypt full message with user key        │
│       - Write ciphertext + iv + authTag to DB     │
│       - If incognito: skip DB write entirely      │
│       - Update token count                        │
│       - Trigger memory extraction (async)         │
└──────────────────────────────────────────────────┘
    │
    ▼
Client Decrypts & Renders
```

### Encryption Implementation

```typescript
// lib/privacy/encrypt.ts

import crypto from 'crypto';

const ALGORITHM = 'aes-256-gcm';

// Derive a unique per-user key from their Clerk user ID + master key
export function deriveUserKey(clerkUserId: string): Buffer {
  return crypto.scryptSync(
    clerkUserId,
    process.env.ENCRYPTION_MASTER_KEY!,
    32
  );
}

export function encrypt(plaintext: string, key: Buffer): {
  ciphertext: string;
  iv: string;
  authTag: string;
} {
  const iv = crypto.randomBytes(16);
  const cipher = crypto.createCipheriv(ALGORITHM, key, iv);
  const encrypted = Buffer.concat([
    cipher.update(plaintext, 'utf8'),
    cipher.final(),
  ]);
  return {
    ciphertext: encrypted.toString('base64'),
    iv: iv.toString('base64'),
    authTag: cipher.getAuthTag().toString('base64'),
  };
}

export function decrypt(ciphertext: string, iv: string, authTag: string, key: Buffer): string {
  const decipher = crypto.createDecipheriv(
    ALGORITHM,
    key,
    Buffer.from(iv, 'base64')
  );
  decipher.setAuthTag(Buffer.from(authTag, 'base64'));
  return Buffer.concat([
    decipher.update(Buffer.from(ciphertext, 'base64')),
    decipher.final(),
  ]).toString('utf8');
}
```

### Anonymizing Proxy Implementation

```typescript
// lib/ai/proxy.ts
// Wraps all outbound API calls to strip identifying information

export function buildAnonymizedHeaders(baseHeaders: Record<string, string> = {}) {
  // Remove any headers that could identify the user
  const { 'x-forwarded-for': _, 'x-real-ip': __, ...safeHeaders } = baseHeaders;
  return {
    ...safeHeaders,
    'User-Agent': 'PrivateAI/1.0',
  };
}
```

## 7. Feature Specifications

### 7.1 Multi-Model Access

**Goal:** Users choose from 7 LLMs organized by task category. Model switching starts a new chat.

#### Available Models

| Model ID | Display Name | Type | Cost Multiplier | Best For |
|---|---|---|---|---|
| `mistral-3b` | Mistral 3B | Open | 0.00x | Simple, ultra-fast queries |
| `mistral-8b` | Mistral 8B | Open | 0.01x | Fast, efficient tasks |
| `qwen-2-5` | Qwen 2.5 | Open | 0.02x | Multilingual, balanced |
| `deepseek-v3` | DeepSeek V3 | Open/API | 0.05x | Reasoning, coding |
| `gemini-pro` | Gemini Pro | Closed | 0.8x | Fast, Google integration |
| `gpt-4o` | GPT-4o | Closed | 1.0x | Broad knowledge, multimodal |
| `claude-3-5-sonnet` | Claude 3.5 Sonnet | Closed | 1.0x | Writing, long context |

#### Model Selector UI Behavior
- Top bar of chat shows currently selected model with cost indicator badge
- Clicking opens a modal/dropdown with category tabs: **Chat**, **Coding**, **Research**
- Each model card shows: name, type badge (Open/Closed), cost multiplier, short strength description
- Switching model triggers a confirmation: *"Starting a new chat with [Model]. Continue?"*
- User can set a per-task default (e.g. "always use Mistral 8B for coding")

### 7.2 Spaces (Contextual Workspaces)

**Goal:** Users create project workspaces where all threads share persistent context, instructions, and memory.

#### Space Object
```typescript
interface Space {
  id: string;
  name: string;                  // Encrypted at rest
  context: string;               // Persistent docs/background — encrypted at rest
  instructions: string;          // How AI behaves — encrypted at rest
```

#### How Context Injection Works
When a new message is sent inside a Space thread, the API route must:
1. Fetch the Space's `context` and `instructions` (decrypt)
2. Fetch the Space's promoted `memory` facts (decrypt)
3. Build a system prompt:
```
[Space Instructions]
{space.instructions}

[Project Context]
{space.context}

[Key Facts from Previous Threads]
- {memory[0].fact}
- {memory[1].fact}
...
```
4. Prepend this system prompt to the messages array sent to the model

#### Cross-Thread Features
- **Thread Linking:** `@thread:<threadId>` syntax in a message fetches that thread's summary and injects it
- **Promote to Space Memory:** Button on any AI message to promote a fact to Space-level memory
- **Unified Search:** `/api/spaces/[spaceId]/search?q=` searches across all encrypted thread messages (decrypt → search → return)

### 7.4 Cross-Context Chat Carryover

**Goal:** Within a chat, the AI always has full context. When a chat hits the model's token limit, a new chat is created with an injected summary so conversation feels seamless.

#### Within-Chat Behavior (Always Active)
These are handled automatically by maintaining the full `messages[]` array in the API call:
- **Conversation Continuity** — full message history sent every request
- **Reference Resolution** — model handles naturally ("Make it shorter" → model knows what "it" is)
- **Correction Memory** — "I prefer bullets" stored in session state and applied to all subsequent messages in that chat
- **Cumulative Understanding** — progressive context builds naturally from message history

#### Token Limit Handling
```typescript
// In app/api/chat/route.ts

const MAX_TOKENS_THRESHOLD = 0.85; // Trigger carryover at 85% of model's context window

async function checkAndHandleTokenLimit(chatId: string, modelId: ModelId, userId: string) {
  const chat = await db.query.chats.findFirst({ where: eq(chats.id, chatId) });
  const maxTokens = MODEL_CONFIG[modelId].maxContextTokens;

  if (chat.tokenCount >= maxTokens * MAX_TOKENS_THRESHOLD) {
    // 1. Generate a summary of the current chat
    const summary = await generateChatSummary(chatId, userId);

    // 2. Store it encrypted in chat_summaries table
    const summaryId = await storeSummary(summary, userId);

    // 3. Create new chat, link carryoverSummaryId
    const newChat = await createNewChatWithCarryover(userId, chatId, summaryId);

    // 4. Return new chat ID so client can redirect
    return { shouldCarryover: true, newChatId: newChat.id, summary };
  }
  return { shouldCarryover: false };
}
```

#### Carryover UI
- Yellow banner at top of new chat: *"Continuing from previous conversation. [Review summary ▾] [Start fresh]"*
- Clicking "Review summary" opens a slide-out panel showing the injected summary
- User can edit/remove items from the summary before proceeding
- "Start fresh" sets `carryoverSummaryId = null` and clears the injected context

### 7.5 Data Redaction (Auto-Privacy Shield)

**Goal:** Automatically detect and redact PII in both user inputs and AI outputs before any processing or storage occurs.

#### Redaction Rules

| Data Type | Pattern / Detection Method | Replacement Token |
|---|---|---|
| Full names | NER model or regex heuristics | `[NAME REDACTED]` |
| Email addresses | Regex: `/[\w.-]+@[\w.-]+\.\w+/g` | `[EMAIL REDACTED]` |
| Phone numbers | Regex: international phone patterns | `[PHONE REDACTED]` |
| Credit card numbers | Luhn check + regex | `[FINANCIAL DATA REDACTED]` |
| SSN | Regex: `\d{3}-\d{2}-\d{4}` | `[FINANCIAL DATA REDACTED]` |
| Home addresses | NER or heuristic patterns | `[ADDRESS REDACTED]` |
| GPS coordinates | Regex: lat/lon patterns | `[ADDRESS REDACTED]` |
| Medical terms in personal context | NER medical entity detection | `[HEALTH INFO REDACTED]` |
| Passport / driver's license | Regex: known formats | `[ID REDACTED]` |
| API keys / tokens / passwords | Regex: key-like patterns | `[SECRET REDACTED]` |

#### Implementation
```typescript
// lib/privacy/redact.ts

const REDACTION_RULES: Array<{ pattern: RegExp; replacement: string }> = [
  { pattern: /[\w.-]+@[\w.-]+\.\w+/gi,          replacement: '[EMAIL REDACTED]' },
  { pattern: /\b\d{3}[-.\s]?\d{2}[-.\s]?\d{4}\b/g, replacement: '[FINANCIAL DATA REDACTED]' },
  { pattern: /\b4[0-9]{12}(?:[0-9]{3})?\b/g,    replacement: '[FINANCIAL DATA REDACTED]' }, // Visa
  { pattern: /\bsk-[a-zA-Z0-9]{20,}\b/g,        replacement: '[SECRET REDACTED]' },          // API keys
  { pattern: /\b[A-Z]{1,2}[0-9]{6,9}\b/g,       replacement: '[ID REDACTED]' },              // Passport
  // Add more patterns
];

export function redactText(input: string): string {
  let output = input;
  for (const rule of REDACTION_RULES) {
    output = output.replace(rule.pattern, rule.replacement);
  }
  return output;
}
```

#### Redaction Settings
- Default: **ON** for all users
- Toggle in Settings: "Auto-redact sensitive information in my inputs"
- Toggle: "Auto-redact sensitive information in AI outputs"
- When toggled off, show a warning modal explaining the privacy implications
- Visual indicator in chat input: small shield icon that turns green when redaction is active

### 7.6 Incognito Mode

**Goal:** Maximum privacy mode. Zero persistence of any kind after session ends.

#### Incognito vs Standard Mode

| Aspect | Standard Mode | Incognito Mode |
|---|---|---|
| Conversation storage | Encrypted and saved to DB | Never written to DB |
| User profile updates | Preferences may be inferred | No profile updates at all |
| Analytics | Anonymous usage metrics (opt-in) | Zero telemetry |
| Context carryover | Available | Disabled — each session isolated |
| Intelligent memory | Active if enabled | Completely disabled |
| Tool history | Cached encrypted | Nothing cached |
| Redis session | Encrypted session stored | Wiped immediately on close |

#### Implementation Rules
1. When `isIncognito = true` on a chat, the `/api/chat` route **must not** write any message to the DB
2. Messages exist only in the client's React state (in-memory) for the duration of the session
3. On tab close / sign-out / 30-minute inactivity: the client clears all state, the server clears the Redis session key
4. No analytics events are fired in incognito sessions
5. The memory engine is bypassed entirely — no reads, no writes

#### Incognito UI
- Persistent dark purple/black top bar when incognito is active: *"🕶 Incognito Mode — Nothing is being saved"*
- Incognito toggle in chat toolbar or sidebar
- On activate: modal explains exactly what incognito does and does not protect
- Model selector still works normally in incognito

### 7.7 Web Search & Deep Research

#### Web Search

| Aspect | Specification |
|---|---|

```typescript
// lib/search/webSearch.ts

export async function webSearch(query: string): Promise<SearchResult[]> {
  const response = await fetch(
    `https://api.search.brave.com/res/v1/web/search?q=${encodeURIComponent(query)}&count=5`,
    {
      headers: {
        'Accept': 'application/json',
        'Accept-Encoding': 'gzip',
        'X-Subscription-Token': process.env.BRAVE_SEARCH_API_KEY!,
        // No user IP, no session token
      },
    }
  );
  const data = await response.json();
  return data.web.results.map((r: any) => ({
    title: r.title,
    url: r.url,
    snippet: r.description,
  }));
}
```

### 7.9 Core Hybrid Architecture

**Goal:** Route queries intelligently between self-hosted open models (max privacy) and closed APIs (max capability), with user control.

#### Routing Decision Logic

```typescript
// lib/ai/router.ts

interface RoutingContext {
  userSelectedModel: ModelId;
  taskType: 'chat' | 'code' | 'research' | 'analysis';
  privacyMaximum: boolean;    // User has set "maximum privacy" preference
  isIncognito: boolean;
}

export function resolveModel(ctx: RoutingContext): ModelId {
  // If user explicitly picked a model, respect that
  if (ctx.userSelectedModel) return ctx.userSelectedModel;

  // If privacy maximum or incognito, use open-source only
  if (ctx.privacyMaximum || ctx.isIncognito) {
    return ctx.taskType === 'code' ? 'deepseek-v3' : 'qwen-2-5';
  }

  // Default routing by task
  const defaults: Record<string, ModelId> = {
    chat:     'gpt-4o',
    code:     'deepseek-v3',
    research: 'gpt-4o',
    analysis: 'claude-3-5-sonnet',
  };
  return defaults[ctx.taskType] ?? 'gpt-4o';
}
```

#### Privacy Guarantee — Always Applied Regardless of Model
- All closed API calls go through `lib/ai/proxy.ts` which strips identifying headers
- User IP is never forwarded to any model provider
- API keys are server-side only, never exposed to the client
- Responses are encrypted before any DB write, regardless of which model generated them

## 9. Privacy & Security Rules

These are **non-negotiable constraints** that apply to every part of the codebase:

1. **No plaintext storage** — Every user-generated content field in the database is AES-256-GCM encrypted. No exceptions.
2. **No IP forwarding** — The proxy layer must strip `x-forwarded-for`, `x-real-ip`, and any fingerprinting headers before all outbound API calls.
3. **No training consent** — All API calls to closed models must use their API-level "no training" flags where available (e.g., OpenAI's `user` field should not include identifying info).
4. **Incognito = no DB writes** — If `isIncognito` is true, no message, summary, or memory fact may be written to the database under any code path.
5. **Keys never leave the server** — Encryption/decryption happens server-side only. The client never sees the user's derived key.
6. **User data is deletable** — Every table that stores user data has `ON DELETE CASCADE` from the `users` table. Account deletion must wipe everything.
7. **No third-party analytics by default** — No Google Analytics, Mixpanel, Hotjar, or similar. If analytics are added, they must be opt-in and anonymized (e.g., Plausible).
8. **API keys are Replit Secrets** — Never hardcode or commit API keys. Access only via `process.env`.
9. **Rate limiting** — All `/api/*` routes must be rate-limited via Upstash Redis to prevent abuse.
