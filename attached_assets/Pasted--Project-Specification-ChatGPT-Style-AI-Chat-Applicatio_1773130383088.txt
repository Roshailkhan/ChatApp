# Project Specification

## ChatGPT-Style AI Chat Application (Frontend Only)

### Platform

Replit

### Objective

Build a complete **ChatGPT-style web application frontend** that allows users to interact with an AI chat backend.

This application must include all frontend logic such as:

* conversation management
* message history
* streaming responses
* context management
* message queue
* retry handling
* error handling
* local persistence

The application **must not implement any backend logic**.

A separate backend team will build the API.

The frontend must be designed so that **only one configuration variable needs to be changed to integrate the backend API**.

---

# Core Integration Requirement

All network requests must use a single configurable endpoint.

Create a configuration file:

`/src/config/apiConfig.js`

Example:

```javascript
export const API_ENDPOINT = "https://backend-domain.com/chat"
```

When backend becomes available, **only this variable should change**.

No other code modifications should be required.

# Context Management

The frontend must automatically manage conversation context.

Algorithm:

1. Collect all conversation messages
2. Preserve system prompts
3. Estimate token size
4. If token limit exceeded:

   * Remove oldest messages
   * Keep latest N messages

This prevents overly large requests.

---

# Context Compression

When conversation becomes extremely long:

Instead of deleting old messages:

1. Generate summary message
2. Replace old messages with summary block

Example:

```
System Summary:
User previously asked about authentication architecture and received an explanation of OAuth flows.
```

---

# Message Queue

Users may send multiple messages rapidly.

Implement a queue system.

Functions:

```
enqueueMessage(message)
processNextMessage()
```

Rules:

* messages processed sequentially
* next message waits until response finishes
* prevents race conditions

---

# Streaming Response Handling

The application must support streaming token responses.

Flow:

```
User sends message
↓
Assistant placeholder message created
↓
Request sent to API
↓
Tokens streamed from server
↓
Tokens appended to message content
↓
UI updates in real time
↓
Message marked complete
```

Streaming must support:

* fetch streaming
* Server Sent Events
* fallback full JSON response

---

# API Client Layer

All API calls must go through:

`/src/services/apiClient.js`

Example:

```
import { API_ENDPOINT } from "../config/apiConfig"

export async function sendChatRequest(payload) {
  return fetch(API_ENDPOINT, {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify(payload)
  })
}
```

No other file should reference the API endpoint.
